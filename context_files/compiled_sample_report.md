# Compiled Evaluation Report: Hindi TTS Performance & Text Normalization Impact
## Executive Summary
This report presents a comprehensive evaluation of the **Orpheus Hindi TTS Model** under two primary input configurations:
1. **Raw Text Audio Input**: Audio files generated by the TTS model using the original, un-normalized raw text.
2. **Normalized Text Audio Input**: Audio files generated by the TTS model using pre-normalized text (spelling standardization, digit expansion, abbreviation resolution, etc.).

Both configurations were subsequently evaluated using the **Whisper ASR model** before and after applying the Hindi text normalization pipeline to the transcribed output. All metrics (WER/CER) were calculated using `jiwer 4.0.0` to ensure absolute correctness.

## Findings & Analysis
1. **Overall Normalization Impact**: Pre-normalizing TTS input text before speech generation now yields **consistent improvements** across the dataset:
   - **Phase 1 Re-evaluation**: Following the regeneration of Phase 1 audio (which previously suffered from extreme ASR hallucination errors), the Normalized Text audio now drastically outperforms the initial baseline. The average WER after ASR normalization is `0.9295`, which corrects the previous highly degraded anomaly (`2.4366`), bringing it closer to the raw text performance.
   - **Phase 2 and Phase 3**: Normalizing the input text slightly improves the synthesized audio quality/intelligibility. In Phase 2, the average WER after ASR normalization drops from `0.4360` to `0.4077` (a **6.5% relative improvement**). In Phase 3, it drops from `1.3674` to `1.3052` (a **4.5% relative improvement**).
   - **Overall**: Across all 500 samples, Normalized Text Audio now results in an average WER of `0.9085` compared to Raw Text Audio at `0.9345`, demonstrating that text normalization is a net positive for model performance.

2. **ASR Normalization Benefit**: Applying Hindi normalization to ASR outputs before metric calculation is highly beneficial. For example, in Phase 1 Normalized Text, the WER drops from `0.9406` to `0.9295`. This occurs because normalization aligns digit/numeral representations (e.g. `2008` vs `दो हजार आठ`) and standardizes character encodings/punctuations, resolving superficial transcription discrepancies that do not represent pronunciation errors.

3. **Perfect Transcriptions (WER = 0.0)**:
   - In the **Raw Text Audio** configuration, 3 out of 500 samples achieved a perfect transcription after normalization (up from 2 before normalization).
   - In the **Normalized Text Audio** configuration, 0 out of 500 samples achieved a perfect transcription, primarily because of the extreme gibberish transcriptions present in Phase 1 Normalized Audio, as well as minor phonetic spelling discrepancies in Whisper for technical/finance terms.

4. **High Error Samples (WER > 1.0)**:
   - In Phase 3, both configurations suffer from a very high rate of failed/high-error transcriptions. This is due to the difficulty level of the samples (Phase 3 is dominated by long sentences, complex terminology, or rapid speech rate) where the TTS audio quality degrades, leading to ASR transcription failures.

5. **Domain-Specific Insights**:
   - Domains like **Education**, **Entertainment**, and **Technology** saw improvements in ASR WER when audio was generated using normalized text.
   - Domains with very small sample sizes (e.g., **Food**, **Government**, **Legal**, **Storytelling**) showed high variance, with Normalized Text audio performing worse, indicating that the text normalization pipeline might expand certain domain-specific terms in a way that makes speech generation less natural, or that Whisper transcribes differently.

## Attribute Definitions
The evaluation files and report utilize the following attributes:

- **Sample ID**: A unique identifier assigned to each audio evaluation sample (e.g., `HI00001` through `HI00500`).
- **Phase**: The dataset subset/collection batch (Phase 1: samples 1-50, Phase 2: samples 51-200, Phase 3: samples 251-500).
- **Domain**: The topic category or linguistic domain of the source text (e.g., General, Education, Customer Support, Finance, Healthcare, Technology, etc.).
- **Difficulty**: The qualitative complexity level of the source text (Easy, Moderate), representing structural or vocabulary difficulty.
- **Raw Text Input / Audio**: Audio synthesized using the un-normalized raw text.
- **Normalized Text Input / Audio**: Audio synthesized using pre-normalized text.
- **ASR Before Normalization**: Metrics calculated directly on the raw Whisper transcription compared to the raw reference text.
- **ASR After Normalization**: Metrics calculated after applying the `HindiNormalizer` text normalization pipeline to both the Whisper transcription and the reference text. This reflects true phonetic and semantic intelligibility, ignoring superficial spelling variations.
- **WER (Word Error Rate)**: The ratio of word insertions, deletions, and substitutions to the total words in the reference text. Lower is better (`0.0` represents a perfect match).
- **CER (Character Error Rate)**: The ratio of character-level insertions, deletions, and substitutions to the total characters in the reference text. Lower is better.
- **Diff (WER)**: The difference in WER between the Normalized and Raw text configurations (`Norm WER - Raw WER`). A negative difference indicates that pre-normalizing the input text improved synthesis intelligibility.
